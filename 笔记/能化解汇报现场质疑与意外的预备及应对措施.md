# 能化解汇报质疑与意外的三个动作

## 预先准备

### 基于汇报内容，掌握其中细节

**汇报细节清单**：

|     分类      | 细节                                                         |
| :-----------: | ------------------------------------------------------------ |
|  目标及背景   | - 客户的需求是什么？<br>- 该方案/项目要解决的问题是什么？<br>- 该方案/项目的目标是什么？拆分出来的小目标有哪些？<br>- 该方案/项目发起的背景是什么？<br>- 该方案/项目需要涉及哪些协作方？哪些部门哪些人？ |
|   潜在风险    | - 该方案/项目实施过程中，可能遇到的障碍有哪些？<br>- 针对这些障碍，准备了哪些解决办法？<br>- 这个方案/项目你都计划哪些资源？（时间、人力、物力、财力）<br>- 这个方案/项目的评价指标是什么？（做到什么程度算做得好） |
|   用户声音    | - 前期做用户调查时，收集到的用户需求是什么？分哪几类？<br>- 具体的数据和依据是什么？排名最靠前的问题是哪些？ |
| 行业/竞品对比 | - 和这个项目类似的业务，竞品、对手公司有哪些？他们是怎么做的？<br>- 他们最近有什么动态？<br>- 行业最近有什么动态？<br>- 和这个项目/业务相关的技术、研究有没有最新进展？ |
|   政府政策    | - 政府最新出台的政策里，哪些是你们部门有关的？<br>- 政府调整/更新了什么政策？可能对这个方案/项目产生的影响是什么？ |
|      ...      | （可持续积累补充）                                           |

**问题预测清单**：

**使用说明**：

1. **先确定参会人员名单**

例如：高层领导来不来、其它部门领导来不来、专家有哪些、法务和财务来不来？

2. **再填写参会人员背景信息和偏好**

可以参考其它同事经验或过往会议记录，总结各个领导的问题偏好。  

因为：人们都喜欢从自己偏好的角度看问题，从而会优先关注自己岗位和专业相关的问题。  

例如：新产品迭代方案汇报过程中：
  - 营销领导可能会问：
    新这个新产品定位和老产品有什么区别？
    据我所知，你的目标用户最关心的应该不是这个问题。你怎么解释？
  - 财务负责人可能会问：
    这个定价为什么要涨价10%？是怎么算出来的？（或者：依据是什么？）
    赠品内容和赠送方式会不会带来额外支出？

3. **后增加问题数量**

可以参考其它同事经验或过往会议记录，总结各个领导的问题数量、问题时长，再据此乘上一个系数（一般3~5，大型可更多）。  

例如：以往这个领导大概率会问2个问题，那么就可对其准备6~10个问题。  

高频问题参考：
  - 营销部门：
      - 新这个新产品定位和老产品有什么区别？
      - 据我所知，你的目标用户最关心的应该不是这个问题。你怎么解释？
  - 财务部门：
      - 这个定价为什么要涨价10%？是怎么算出来的？（或者：依据是什么？）
      - 赠品内容和赠送方式会不会带来额外支出？
      - 特殊用户的定义是什么？
  - 销售部门：
      - 这个产品的卖点是什么？跟之前的有什么区别？
      - 这个产品有什么服务吗？
      - 给用户的福利政策有点复杂，为什么这么设计？
  - ...（可持续积累补充）

## 应对质疑

**注意：面对提问，切忌站到领导对立面。**
  - **说明**：
      - 应对质疑所作的预先准备并非用来在汇报现场与人辩论抬杠，
      - 而是要证明你的想法与专家领导是一致的、想一块了、心流了。
  - **反例：特别容易出现**：
      - “哎，领导不是这样的。”
      - “哎，领导，我给你解释解释……”
      （领导的心理活动：咋了，我错怪你了，我说错了呗？）

**要点：回应质疑 = 承接情绪 + 导向行动 。**
  - **第一步：承接情绪。积极回应领导和专家的关切提问。**
    **如果**自己对问题答案有自信，可说：
      - “领导，您问的这个问题，刚好就是我们内部开了5次会讨论出来的一个关键争议点。”
      - “领导您说的这个点确实也是我们的一个关键创新点……”
    **如果**自己对问题答案没自信，可说：
      - “谢谢领导，您提的这个问题确实很重要，指出了我们方案里的不足。”
    **如果**领导是个急脾气，不爱听解释，就先重点说结论：
      - “领导，您说的这件事我们也关注到了，我掌握的情况跟您刚刚的结论是一致的。”
      - “领导，确实像您说的，客户的需求也是这样……”
    **如果**领导是技术专家出身，特别严谨细致，那就先重点说细节：
      - “谢谢领导，这个技术点确实很重要。接下来我方案里补充一下……”
      - “领导我理解您的担心，这里确实有一个容错偏差需要讨论……”
    **如果**对方平时很注重关系感受，喜欢被看到，那就要兼顾情绪价值：
      - “您说的这个太重要了，辛亏有您提醒。”
      - “您提的这点确实是一阵见血……”
        （注意点到为止，切忌用力过猛）
  - **第二步：导向行动。针对问题给出接下来的行动信息。**
    **例如**：
      小王正在推进一个新的项目，返工了好几次。隔壁部门的人不仅是等着看笑话，还故意调侃说：“我们都跟着你熬了好几个大夜了，这个项目可不能折了啊。”同事说这样的话其实是带着情绪的。
    **应对**：
      - **首先承接情绪**：“我知道大家最近很辛苦，我也听到了一些不同的声音，这些声音我都接受，说得有道理。所以我专门把大家请过来，赶紧汇报一下最新的优化方案。”
      - **接着导向行动**：“我已经详细分析了出问题的原因，并且制定了改正措施。第一点分别是什么，第二分别是什么。另外我也专门跟领导做了申请，争取到了什么什么资源……”
    **又如**：
        要给大领导汇报方案了，结果你的直属领导突然有事儿来不了，就你最了解情况，所以临时喊你去补位。可你去了现场还没开口，大领导就问你：“你领导自己都不来，是不是觉得这个项目不重要啊？”那这个时候咱肯定不能说：“啊，不是的领导，我领导有其他事才派我来的。”
    **应对**：
      - **首先承接情绪**：“领导，张总也一直担心，说怕影响不好，所以专门让我先跟大家道个歉。他现在正在X公司堵着他们老板聊折扣价格，让我们等他的好消息。”
      - **接着导向行动**：“那今天呢，汇报完，我也会记录好各位领导的建议，回去一定同步落实到位。”

注：领导的提问质疑，其实就是在讨论对齐工作，不是针对某个人。他只是想确定，你能不能告诉我到底是什么情况？你到底想清楚了没有？他其实是在考察和验证你的工作方案。只要你能给出完备的解答，那么领导的这个质疑就会收回去了。

## 应对意外

**要点：应对突发意外 = 放弃旧规则 + 引入新规则 。**

**例如**：显示屏突发故障，无法正常播放PPT或材料。
**应对**：
  - 第一句话，放弃旧规则：“哎，这显示屏出问题了，那咱就不看大屏幕了。”
  - 第二句话，开始新规则：“刚好这儿又一个白板，那我就一边说以便记录。这样我们看着白板说……”
  - 如果条件允许，可以提前打印材料发给每位汇报对象，那如果出现这种情况，就可以说：“大家可以看手边的资料，那现在我们说到了第几页，第几条……”

**例如**：话筒突发故障，无法正常发声。
**应对**：
  - 第一句话，放弃旧规则：“哎，这话筒出问题了，那咱就不用话筒了。”
  - 第二句话，开始新规则：“接下来我大声点说。好，那刚刚呢我们说到了方案的第二个部分……”
  - 如果现场人特别多，可以走到人群中欧国际那，然后说：“为了让大家都尽量能够听清楚，那我就走到中间来说。那前排的同事可能比较辛苦，如果有谁没听清楚，可以随时打断我。好，那下面我接着讲……”

## 总结

应对质疑和突发情况，关键就是：
  - 前：掌握细节，准备预案；
  - 中：不要闪躲，正面回应；
  - 后：导向行动，启用新规则。
