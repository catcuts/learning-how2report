# 让领导提供支持并表态的汇报结尾设计

汇报的最终目的是导向行动，领导没表态=行动难落实。

公式：
汇报结尾 = 站点句 + 垫一句 + 导行动。

示例：
    小王要负责一款金融产品的升级，他给领导汇报了一下这款产品的升级方案。结尾是这么说的：“我的汇报到此结束，未来落地方案的时候，还要麻烦大家多多支持，感谢感谢。”
    看起来挺诚恳了是吧？好像也导向了行动——希望大家多多支持。但我告诉你，这是一个假的导向行动，这更像是一个单方面的许愿，就算是现场有人想要支持你，他都不知道该从哪儿下手。
    那怎么改呢？来，我们按照“站点句+垫一句+导行动”的方式就可以这么说：
    1. 【站点句】“好，以上就是我关于产品升级的初步方案。”
        这是站点句。意思是：注意了，主要内容我已经介绍完了，马上就要结束了。这样就可以把现场人的注意力都拉回来。
    2. 【垫一句】“总之，通过升级，我们很有信心，新用户增长预计将增加18%，而产生的收益将提高25%。”
        接着垫一句。如果你对方案很有信心，那就可以再次把方案可能带来的好处、收益强调一遍，争取现场人的认可。这样后面咱们导向行动的时候才有依据，有底气。
    3. 【导行动】“为了让这个升级方案得到顺利的实施，我们需要增加一个产品的开发人员以及 5 个资源位，用于后期的市场推广，这也是我们内部讨论后的最小化的需求，所以希望能够得到领导的建议和支持。”
        最后导行动。是想要跟领导申请资源，还是希望领导给点儿建议，这个时候就可以直接说。

## 站点句

要点：无需花样，只需停顿+音量。
示例：先停顿两秒钟，然后提高音量，把声音放出来：
    “好，以上就是我要汇报的主要内容了……”

# 垫一句

要点：垫一个能驱动领导表态的理由。包括：
  1. 基于亮点，重点展示你的信心、强调收益和好处。
     例如：“总之，我们很有信心通过这个方案，把客户满意度提高到95%，年度收入实现 10% 的增长。”
  2. 基于卡点，放下心理包袱、呈现出问题不解决、后果很严重的姿态。
     例如：“确实在项目的推进中，我们遇到了什么什么技术难题，但如果这个问题不解决，咱们降低 40% 成本、提升 30% 效率的目标就没办法达成了。”

# 导行动

要点：说明领导和协作者具体要做什么。

如果项目较简单、方案较明确，那么可以直接说：“最后再请领导确认一下这个方案是否可行。”
如果涉及部门多、方案较复杂，那么需要分情况：

第一种情况：如果需要解决问题，那么就针对问题展开讨论，让在场的人都有话可说。
例如：“现在这个推广方案最大的卡点就是渠道比较单一。今天机会难得，各位领导都在，我想一起讨论一下这个新渠道拓展方式是否可行，可以的话，我们下去就继续推进了。”

第二种情况：如果需要获得资源，那么就针对资源发起请求，注意成本勿高要求勿多。
例如：“这个项目的落实离不开咱们业务、中台和技术部门同事的支持。为了让项目尽快地走上正轨，我想再多要两个支持：一个是，您看能不能让业务部门，给我们做一次培训指导，方便我们快速上手。另外一个就是，我们能不能要您一点时间，每周不多，就花您30分钟，给我们专门指导一下，听听我们的汇报。”

第三种情况：如果需要配合工作，那么就针对工作指派任务，最好是列一个待办清单（包括分工内容、负责人、完成时间等，确保每个人都知道各自的职责和标准）。
例如：“好，既然方案都已经确定了，那我们马上开始往下推进，我再次对齐一下，产品开发的负责人是小王，小李和小杨负责协同，预计完成的时间需要是在什么什么时间。内测这边的负责人是……”
